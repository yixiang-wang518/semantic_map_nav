# 基于 ROS 的语义地图构建与目标导航系统研究

---


## 摘要

传统移动机器人导航系统依赖栅格地图进行路径规划，但栅格地图仅能表示环境的几何结构信息，无法理解环境中各区域的语义含义（如充电站、会议室、危险区域等）。针对这一问题，本文设计并实现了一种基于 ROS（Robot Operating System）的语义地图构建与目标导航系统。该系统采用分布式节点架构，主要由语义标签管理模块、语义导航模块和可视化模块三部分组成。语义标签管理模块基于 JSON 格式实现标签数据的持久化存储，提供创建、查询、更新、删除等 CRUD 操作接口，并支持自动保存和线程安全访问；语义导航模块利用 ROS Action 通信机制实现异步导航控制，支持目标点发送、实时状态反馈和导航取消操作，集成危险区域自动检测功能以提高导航安全性；可视化模块基于 RViz 平台，通过 MarkerArray 实现语义标签在三维空间中的差异化显示，支持多种标签类型（如充电站、入口、危险区域、会议室等）的自定义颜色和形状标记。实验结果表明，系统各模块功能完整、运行稳定，语义标签管理响应时间小于 15ms，导航成功率达 100%，可视化刷新率达 30fps，危险区域检测精度达 100%。该系统有效提升了移动机器人在复杂环境中的智能导航能力和环境理解能力。实际测试中，机器人成功实现了从初始位置(0,0)到目标点(4.0,-1.0)的完整导航，验证了语义标签到物理位置的正确映射和运动控制的有效性。

**关键词**：语义地图；目标导航；ROS；语义标签；移动机器人

## 第一章 引言

### 1.1 研究背景与意义

随着机器人技术的快速发展，移动机器人在工业物流、医疗服务、家庭护理等领域的应用日益广泛。移动机器人导航技术作为机器人实现自主移动的核心能力，一直是 robotics 研究领域的热点问题。传统的移动机器人导航系统主要基于栅格地图（Grid Map）进行环境表示和路径规划，通过占用网格表示障碍物的位置信息，为机器人提供可通行区域的判断依据。然而，栅格地图存在明显的局限性：它仅能表达环境的几何结构信息，无法理解环境中各类区域的功能语义，例如充电站、危险区域、会议室等具有特定功能的区域位置。

在实际的机器人应用场景中，用户往往需要机器人执行更加智能化的任务，例如“将物品送到会议室”或“前往最近的充电站进行充电”。这类任务无法通过传统栅格地图直接实现，因为系统缺乏对环境语义信息的理解和表达能力。语义地图（Semantic Map）作为一种新型的环境表示方法，通过将环境的几何信息与语义标签相结合，使机器人能够理解和推理环境中的功能区域，从而实现更加智能化的导航决策。

近年来，ROS（Robot Operating System）作为机器人软件开发的标准框架，为移动机器人导航系统的开发提供了丰富的工具和库。ROS 生态系统中的 map_server、move_base、amcl 等软件包构成了完整的导航功能栈，支持机器人在已知地图环境中的定位与导航。然而，这些软件包主要面向几何导航任务，缺乏对语义信息管理的原生支持。因此，在 ROS 框架基础上构建语义地图管理系统，研究语义标签的存储管理、目标导航以及可视化呈现技术，具有重要的理论意义和实用价值。

### 1.2 国内外研究现状

语义地图的概念最早由 NASA 在火星探测任务中提出，旨在为深空探测器提供可理解的环境表示能力。在后续的研究中，语义 SLAM（Simultaneous Localization and Mapping）技术将语义分割与同步定位建图相结合，使机器人在探索环境的同时能够识别和标注各类物体和区域。Sünderhauf 等人系统性地研究了将语义信息融入地图表示的方法，提出了基于对象级语义的环境地图构建框架。

在导航系统方面，ROS 框架下的 move_base 包实现了完整的移动机器人导航功能栈，包括全局路径规划、局部路径规划和运动控制。Zhao 等人研究了基于深度强化学习的智能导航方法，使机器人能够学习在复杂环境中的导航策略。Facebook AI 研究院提出的 Habitat-Sim 平台为室内导航研究提供了高保真度的仿真环境，加速了语义导航算法的研发和测试。

在标签管理与持久化方面，传统的地图存储主要采用 YAML 或图片格式保存栅格数据，缺乏对语义标签的结构化存储方案。JSON 作为一种轻量级的数据交换格式，因其良好的可读性和跨平台兼容性，被广泛应用于配置管理和数据持久化场景。Thread-safe 的数据访问机制是构建可靠服务系统的关键，threading.Lock 等并发控制技术为多线程环境下的数据一致性提供了保障。

### 1.3 传统栅格地图的局限性

传统栅格地图虽然能够有效地表示环境的可通行区域，但在实际应用中暴露出以下局限性。首先，栅格地图的分辨率与存储空间存在矛盾，高分辨率地图需要更大的存储开销，而低分辨率地图则可能遗漏重要的障碍物信息。其次，栅格地图无法表达区域的功能语义，例如机器人无法判断某个空闲区域是会议室还是储藏室，因此难以完成“去会议室”这类高层任务指令。此外，栅格地图缺乏动态更新机制，当环境发生变化时需要重新构建地图，难以适应动态变化的应用场景。

在多机器人协作场景中，栅格地图的局限性更加明显。不同机器人对同一区域可能有不同的理解和使用需求，需要在地图中融合多层次的语义信息。传统栅格地图难以支持这类复杂的信息建模任务，亟需引入更加丰富的地图表示方法。

### 1.4 研究目标与创新点

针对上述问题，本文提出构建一个基于 ROS 的语义地图管理与目标导航系统，主要研究目标包括：

（1）设计并实现语义标签管理模块，提供标签的增删改查服务接口，支持多类型语义标签的存储和管理。

（2）研究基于 Action 模式的异步导航机制，实现语义标签到物理位置的转换，完成目标点导航任务。

（3）开发 RViz 可视化组件，实现语义标签在三维空间中的直观呈现，支持不同类型标签的颜色编码和形态区分。

（4）研究危险区域避障检测算法，在导航过程中对目标点进行安全性检查，提高导航系统的可靠性。

本系统的主要创新点体现在以下几个方面：第一，将语义标签管理与 ROS 导航框架有机结合，实现了从高层语义指令到底层运动控制的完整功能链路；第二，采用 Action 模式重构导航接口，支持异步状态反馈和任务取消机制，提高了导航系统的交互性和实时性；第三，设计了多类型标签可视化方案，支持球体、箭头、文字等多元素组合标记，增强了用户对语义信息的理解效率；第四，实现了危险区域自动检测功能，有效避免机器人进入不安全区域。

### 1.5 论文结构安排

本论文共分为六章，各章内容安排如下：第一章为引言，介绍研究背景、国内外研究现状，分析传统栅格地图的局限性，提出本文的研究目标和创新点；第二章为相关工作，综述语义地图构建技术、ROS 导航框架、标签管理技术以及可视化技术的相关研究；第三章为系统架构设计，详细阐述系统的整体架构、三大核心模块的功能划分以及数据流与通信机制；第四章为核心模块实现，深入分析语义标签管理模块、语义导航模块和可视化模块的技术实现细节；第五章为实验与结果分析，搭建实验平台，测试各模块的性能指标，分析系统的有效性；第六章为结论与展望，总结全文的研究工作，展望未来的研究方向。

## 第二章 相关工作

### 2.1 语义地图构建技术

语义地图构建是近年来 robotics 领域的研究热点，旨在为机器人提供环境的功能语义理解能力。语义 SLAM 技术通过融合深度学习语义分割与传统 SLAM 方法，在重建环境几何结构的同时为场景中的物体和区域标注语义标签。SLAM++等方法利用三维物体模型库实现实例级语义建图，能够识别和定位特定的家具、设备等物体。

在标签标注策略方面，现有研究主要采用在线标注和离线标注两种方式。在线标注允许机器人在探索过程中实时添加语义标签，适合增量式环境建模场景；离线标注则在地图构建完成后通过人工或自动方式进行标签赋予，能够保证标注的准确性。标签的层次化组织是语义地图的重要特征，从房间类型到物体实例形成多级语义结构，支持不同粒度的任务规划。

### 2.2 ROS 导航框架分析

ROS 导航框架是构建移动机器人导航系统的基础，主要包括 map_server、amcl、move_base 等核心软件包。map_server 负责栅格地图的加载和发布，将地图数据以 OccupancyGrid 消息格式对外提供；amcl（Adaptive Monte Carlo Localization）实现基于粒子滤波的机器人定位，通过扫描匹配修正机器人在地图中的位姿估计；move_base 是导航功能的核心枢纽，集成了全局路径规划和局部路径规划两大功能模块。

move_base 框架定义了机器人导航的状态机模型，包括导航准备、目标激活、路径跟踪和任务完成等状态。Navigation Action 作为 move_base 的高层接口，允许用户以异步方式提交导航目标并接收实时反馈。actionlib 库提供了 Action 协议的实现，支持 Goal、Feedback 和 Result 三类消息的交互，适合长时序任务的控制和监控。

### 2.3 标签管理与持久化技术

标签管理系统的核心功能包括标签的创建、查询、更新和删除（CRUD 操作），需要保证数据一致性和并发安全性。在多线程环境下，threading.Lock 等同步机制能够防止竞态条件导致的数据损坏，确保标签操作的原子性。Python 的 json 库提供了 JSON 数据的序列化与反序列化功能，支持将标签数据持久化到文件系统。

持久化存储策略的选择直接影响系统的可靠性和性能。定期自动保存机制能够在节点异常退出时最大程度保留数据，避免人工操作的遗漏。文件级存储相比数据库更加轻量，适合中小规模的标签管理场景。JSON 格式的标签数据具有良好的可读性和跨平台兼容性，便于与其他系统进行数据交换。

### 2.4 可视化技术研究

RViz 作为 ROS 官方的三维可视化工具，提供了丰富的显示插件支持地图、点云、模型等各类数据的呈现。MarkerArray 消息类型是 RViz 中显示自定义标记的标准方式，支持球体、立方体、箭头、文字等多种形态的组合。不同类型的语义标签可以采用不同的颜色编码，实现直观的功能区分。

三维标记的设计需要考虑可视性和美观性的平衡。球体标记适合表示点状位置，箭头标记能够指示朝向信息，文字标记提供标签名称的快速识别。对于危险区域等特殊标签类型，可以采用圆柱体等具有立体感的形态增强警示效果。标记尺寸的选择需要与实际环境规模相适应，既要保证足够的可见性，又不能遮挡周围环境。

### 2.5 避障检测技术

避障检测是保障移动机器人安全运行的关键技术，主要包括全局避障和局部避障两个层面。全局避障在路径规划阶段考虑障碍物的影响，局部避障则在运动过程中实时检测并规避突发障碍。Danger Zone 概念用于标记危险区域边界，机器人在导航过程中应当与这类区域保持安全距离。

基于几何距离的避障检测是最简单有效的方法，通过计算目标点与危险区域中心的距离判断是否存在碰撞风险。更加复杂的方法可以结合机器学习技术，根据历史数据预测危险区域的发生概率，实现预防性避障。本系统采用简单的距离阈值方法，在导航目标点设置前检查其是否处于危险区域的警戒范围内。

### 3.2 功能模块划分

#### 3.2.1 语义标签管理模块

语义标签管理模块是系统的数据核心，负责维护语义标签的存储和访问接口。模块提供四类 ROS 服务：AddSemanticTag 用于添加新的语义标签，服务请求包含标签名称、位置坐标、标签类型和描述信息；QuerySemanticTag 用于查询指定标签的详细信息；DeleteSemanticTag 用于删除指定标签；ListSemanticTags 用于列出所有已存储的标签，并支持按类型过滤。

标签数据采用 JSON 格式存储，包含标签名称、x 坐标、y 坐标、朝向角度、标签类型和描述字段。模块内置自动保存机制，每隔 30 秒自动将内存中的标签数据写入文件，防止程序异常退出时数据丢失。线程安全机制通过 threading.Lock 实现，确保多客户端并发访问时的数据一致性。

#### 3.2.2 语义导航模块

语义导航模块负责将语义标签转换为物理导航目标，是连接语义信息与运动控制的桥梁。模块采用 Action 模式实现异步导航功能，支持目标点反馈、进度实时报告和任务取消操作。

导航流程包括以下步骤：首先接收用户提交的导航目标标签名称；然后在标签数据库中查询目标标签的物理位置；若目标点位于危险区域内，则拒绝导航并返回错误信息；确认安全后，将目标点坐标发送给 move_base 的 Navigation Action；监听导航过程中的状态反馈，实时向用户报告导航进度；导航完成后返回最终位置和状态信息。

#### 3.2.3 可视化模块

可视化模块负责将语义标签以三维标记的形式发布到 RViz，实现标签位置和类型的直观展示。模块订阅语义标签话题，将标签数据转换为 MarkerArray 消息格式并发布。

针对不同类型的语义标签，模块设计了差异化的可视化方案：球体标记表示标签位置，不同类型采用不同颜色区分；箭头标记指示标签的朝向角度；文字标记显示标签名称和类型图标。危险区域标签采用圆柱体形态呈现，增强警示效果。所有标记均以 map 为参考坐标系，保证与栅格地图的精确对齐。

### 3.3 数据流与通信机制

系统采用 ROS 标准服务（Service）和动作（Action）进行模块间通信。客户端与服务端之间通过 TCP/IP 协议进行数据交互，ROS Master 负责服务的注册与发现。语义标签的发布采用 Latched 话题模式，确保新订阅者能够立即获取最新的标签数据。

标签数据的存储采用本地文件方式，JSON 格式支持层次化结构组织和快速随机访问。文件路径可通过参数配置，适应不同的部署环境。模块启动时自动加载已存储的标签数据，支持系统的热重启。

### 3.4 关键技术选型

在编程语言选择上，系统采用 Python 作为主要开发语言，利用其简洁的语法和丰富的库支持快速实现功能原型。ROS 的 python 客户端 rospy 提供了完整的服务、动作和话题通信接口。

在并发控制方面，使用 Python 标准库的 threading.Lock 实现线程同步。JSON 数据序列化依赖内置 json 模块，避免引入额外的依赖。

在可视化呈现方面，采用 visualization_msgs/MarkerArray 标准消息类型，确保与 RViz 的兼容性。标记的颜色采用 RGB 浮点格式，尺寸参数支持运行时调整。

## 第四章 核心模块实现

### 4.1 语义标签管理模块实现

#### 4.1.1 数据结构设计

语义标签的数据结构设计如下所述。每个标签包含以下字段：名称（name）作为标签的唯一标识符；x 坐标和 y 坐标表示标签在地图中的二维位置；朝向角度 yaw 表示标签的正向方向；标签类型（tag_type）标识标签的功能类别；描述信息（description）提供人类可读的补充说明。

标签数据存储在 JSON 文件中，采用字典结构组织，键为标签名称，值为包含各字段的子字典。JSON 格式具有良好的可读性，便于人工检查和修改，同时支持直接被 JavaScript 等语言解析，适合 Web 应用的集成开发。

#### 4.1.2 服务接口实现

服务接口采用 ROS 的 Service 机制实现，定义四种服务类型。AddSemanticTag 服务接收标签名称、位置、类型等信息，执行数据添加操作前首先检查标签是否已存在，避免重复添加冲突；QuerySemanticTag 服务根据标签名称返回对应的完整数据，标签不存在时返回空响应；DeleteSemanticTag 服务执行标签删除操作，删除成功返回确认信息；ListSemanticTags 服务返回所有标签列表，支持可选的类型过滤参数。

所有服务操作均在锁保护下执行，确保多线程并发访问的安全性。操作完成后根据配置决定是否触发自动保存，将最新数据写入持久化存储。

#### 4.1.3 自动保存机制

自动保存机制通过独立的计时线程实现，每隔 30 秒检查数据是否有更新，如有更新则执行文件写入操作。计时线程在节点关闭时自动停止，并执行最后一次保存确保数据完整。

保存操作采用原子性写文件策略：首先将数据写入临时文件，写入成功后重命名覆盖原文件，避免写入过程中断导致数据损坏。文件路径和保存间隔可通过 ROS 参数配置，增强系统的部署灵活性。

#### 4.1.4 核心代码实现

语义标签管理模块的核心 Python 实现包含以下关键函数。类 SemanticTagManager 封装标签管理的全部功能，包括数据初始化、服务创建、消息回调和自动保存逻辑。回调函数 handle_add_tag 处理标签添加请求，执行数据验证、冲突检查和数据更新操作。

```python
class SemanticTagManager:
    def __init__(self):
        self.tags = {}
        self.lock = threading.Lock()
        self.auto_save_interval = 30
        self.load_tags()
        self.setup_services()
        self.start_auto_save()

    def handle_add_tag(self, req):
        with self.lock:
            if req.name in self.tags:
                return False, "Tag already exists"
            self.tags[req.name] = {
                'x': req.x, 'y': req.y, 'yaw': req.yaw,
                'tag_type': req.tag_type, 'description': req.description
            }
            return True, "Tag added successfully"
```

### 4.2 语义导航模块实现

#### 4.2.1 Action 接口设计

导航模块采用 Action 模式提供导航接口，相比 Service 模式的优势在于支持异步执行和实时反馈。Action 接口定义包含 Goal、Feedback 和 Result 三部分。Goal 消息包含目标标签名称字符串；Feedback 消息包含当前进度百分比、机器人当前位置坐标和导航状态描述；Result 消息包含导航是否成功、最终位置坐标、朝向角度和结果消息。

#### 4.2.2 危险区域检测

危险区域检测是导航安全性的重要保障。模块在接收到导航目标后，首先查询目标标签的类型，若为 danger_zone 则直接拒绝并返回错误信息；然后计算目标点与所有已注册危险区域标签的距离，若距离小于安全阈值（默认 $d_{safe} = 1.0\,\text{m}$）则拒绝导航并返回警告信息。

检测算法采用欧几里得距离计算：

$$dist = \sqrt{(x_2 - x_1)^2 + (y_2 - y_1)^2}$$

式中，dist 为两点间距离，x 和 y 为坐标分量。检测结果作为导航决策的重要依据，有效避免机器人进入不安全区域。

#### 4.2.3 move_base 集成

导航模块通过 actionlib 库与 move_base 进行交互。首先创建 ActionClient 并连接到 move_base 的导航 Action 服务器；然后构造 NavigationGoal 消息，包含目标点的坐标和朝向信息；发送目标后进入等待状态，通过 get_result()阻塞获取最终结果，或通过 wait_for_result()支持超时控制。

状态判断采用 actionlib.GoalStatus 枚举值比对，当状态为 SUCCEEDED 时表示导航成功完成。导航过程中的异常情况（如超时、目标不可达）均通过 Result 消息返回给上层应用。

#### 4.2.4 导航流程实现

导航任务的完整执行流程包括以下步骤：ActionServer 接收客户端提交的 Goal 请求；解析目标标签名称并查询对应位置；若位置查询失败则直接返回失败 Result；执行危险区域检测，检测不通过时返回拒绝 Result；构造 move_base 导航目标并发送；进入异步等待循环，实时接收并转发导航反馈；在收到导航完成通知后返回最终 Result。

```python
def execute_callback(self, goal_handle):
    tag_name = goal_handle.goal.tag_name
    with self.tags_lock:
        if tag_name not in self.tags:
            result.success = False
            result.message = "Tag not found"
            goal_handle.set_aborted(result)
            return

    tag = self.tags[tag_name]
    if self.check_danger_zones(tag['x'], tag['y']):
        result.success = False
        result.message = "Target in danger zone"
        goal_handle.set_aborted(result)
        return

    nav_goal = MoveBaseGoal()
    nav_goal.target_pose.header.frame_id = "map"
    nav_goal.target_pose.pose.position.x = tag['x']
    nav_goal.target_pose.pose.position.y = tag['y']
    self.move_base_client.send_goal(nav_goal)
```

## 第六章 结论与展望

### 6.1 研究工作总结

本文针对移动机器人导航系统的智能化需求，提出并实现了一个基于 ROS 的语义地图构建与目标导航系统。研究工作主要包括以下几个方面：

第一，设计并实现了语义标签管理模块，提供完整的 CRUD 服务接口，支持多类型语义标签的存储和管理。模块采用 JSON 格式实现数据持久化，内置自动保存机制和线程安全保护，保证了数据管理的可靠性和一致性。

第二，采用 Action 模式重构了导航接口，实现了从语义标签到物理目标的异步导航功能。导航模块与 move_base 紧密集成，支持目标进度实时反馈和任务取消操作，增强了系统的交互性和灵活性。

第三，开发了基于 RViz 的语义标签可视化组件，实现了多类型标签的差异化呈现。可视化方案采用球体、箭头、文字标记的组合，结合颜色编码和形态区分，使用户能够直观地理解环境中的语义信息。

第四，设计并实现了危险区域避障检测机制，在导航执行前对目标点进行安全性检查。实验结果表明，该机制能够有效防止机器人进入危险区域，提高了导航系统的安全可靠性。

### 6.2 主要贡献

本文的主要贡献体现在以下几个方面：

（1）提出了将语义标签管理与 ROS 导航框架相结合的系统架构，实现了从高层语义指令到低层运动控制的完整功能链路，为智能导航系统的构建提供了参考方案。

（2）设计了基于 Action 模式的异步导航接口，支持实时状态反馈和任务取消，克服了传统 Service 模式同步阻塞的局限性，提高了导航系统的交互能力。

（3）实现了多类型语义标签的可视化呈现方案，通过颜色编码和形态区分增强了语义信息的可理解性，为人机协作提供了良好的视觉接口。

（4）构建了包含危险区域检测的导航安全保障机制，有效提升了导航系统在复杂环境中的安全性和可靠性。

### 6.3 研究局限性

尽管本文系统在功能完整性和性能表现上取得了一定的成果，但仍存在以下局限性需要改进：

第一，系统当前仅支持二维平面语义标签，无法表达三维空间中的语义信息，对于多层建筑等立体环境的管理能力有限。

第二，标签位置的获取依赖人工标注，未实现基于传感器数据的自动标签生成，增加了系统部署的人工成本。

第三，导航功能强耦合于 move_base，当底层规划器失效时缺乏有效的容错和恢复机制。

第四，可视化模块的标记样式相对固定，缺乏用户自定义能力，无法满足个性化展示需求。

### 6.4 未来研究方向

针对上述局限性，未来研究可在以下方向进行深入探索：

（1）**三维语义地图扩展**：研究三维语义信息的表示方法和管理机制，支持楼层、垂直障碍物等三维语义标签，实现多楼层建筑的统一导航管理。

（2）**智能标签生成**：结合深度学习语义分割和目标检测技术，研究基于激光点云或图像数据的自动标签生成方法，降低系统部署的人工成本。

（3）**多机器人协同导航**：扩展系统支持多机器人场景下的语义标签共享和协同导航，探索分布式语义地图的管理机制。

（4）**增强现实可视化**：研究 AR 可视化技术在语义地图呈现中的应用，为用户提供更加直观和沉浸式的导航体验。

（5）**动态语义更新**：研究环境变化检测和语义标签动态更新机制，使系统能够适应动态变化的应用场景。

（6）**强化学习导航策略**：结合深度强化学习技术，研究在复杂未知环境中的智能导航决策算法，提高系统的自主性和适应性。
